# Sprint 8. Итоговый проект — Задание 2: API

## Описание
Автоматизация тестирования REST API учебного сервиса «Stellar Burgers». 
Покрыты основные эндпойнты: создание, логин, изменение данных пользователя, создание заказа и получение заказов конкретного пользователя.

## Тестовые сценарии

### 1. Создание пользователя
-  создать уникального пользователя;
-  создать пользователя, который уже зарегистрирован;
-  создать пользователя и не заполнить одно из обязательных полей.
### 2. Логин пользователя
-  логин под существующим пользователем;
-  логин с неверным логином и паролем.
### 3. Изменение данных пользователя
-  изменение данных с авторизацией;
-  изменение данных без авторизации.
### 4. Создание заказа
-  создание заказа с авторизацией;
-  создание заказа без авторизации;
-  создание заказа с ингредиентами;
-  создание заказа без ингредиентов;
-  создание заказа с неверным хешем ингредиентов.
### 5. Получение заказов конкретного пользователя
-  получение заказов авторизованного пользователя;
-  получение заказов неавторизованного пользователя.

## Запуск тестов

1. Клонировать репозиторий
```bash
git clone https://github.com/KBulat/Diplom_2.git
```

2. Установить зависимости
```bash
pip install -r requirements.txt
```

3. Запустить тесты
```bash
pytest -v --alluredir=allure_results
```

4. Просмотреть отчёт о тестировании
```bash
allure serve allure_results
```
